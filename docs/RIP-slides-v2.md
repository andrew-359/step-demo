# Слайды для презентации директору v3
## "Карта связей" — система анализа проектов

> Аудитория: директор строительной фирмы, технически не подготовлен
> Цель: продать идею, снять страхи, получить добро на PoC
> Тон: простой, конкретный, без терминов
> Слайдов: 8 (0–7)

---

## Слайд 0 — Проблема

**Заголовок:** "Каждый проект — поиск с нуля"

**Три пункта:**
- Знания о нужных людях хранятся в головах менеджеров
- Уходит менеджер — уходят знания
- Под каждый новый проект всё собирается заново вручную

**Ключевая фраза:**
> "Вы уже работали с этими людьми. Просто не помните."

---

## Слайд 1 — Главная идея

**Заголовок:** "Корпоративная память о связях"

**Диаграмма:** граф рукопожатий с путём до ЛПР (draw.io, how-what-we-get)

**Три тезиса:**
- Не CRM и не адресная книга — карта рукопожатий
- Каждый проект пополняет общую память компании
- Связи компании — это её самый ценный нематериальный актив

**Ключевая фраза:**
> "У вас крупные заказы. Одна найденная связь — это контракт."

---

## Слайд 2 — Как это работает

**Заголовок:** "Система + человек — вместе точнее"

**Диаграмма:** четыре блока, чередование менеджер/система (draw.io, how-to-use)

**Четыре шага:**
```
ШАГ 1 — Менеджер
Создаю проект. Добавляю компанию заказчика по ИНН.

ШАГ 2 — Система
Автоматически обогащаю данные из реестров.
Нахожу руководителей, учредителей, связанные компании.
Предлагаю добавить найденных людей в граф.

ШАГ 3 — Менеджер
Проверяю. Исправляю что система поняла неверно.
Добавляю что знаю сам — контекст, тип связи, кто ЛПР.

ШАГ 4 — Система
Граф готов. Генерирую отчёт:
карта связей + стратегии входа + оценка риска каждого пути.
```

**Важный момент про вероятности:**
Система не говорит "директор — Иванов". Она говорит "скорее всего директор — Иванов, источник реестр". Менеджер видит уверенность системы и принимает финальное решение сам.

**Ключевая фраза:**
> "Система предлагает — менеджер решает. Знания остаются в компании навсегда."

**Заметка докладчику:**
Директор может бояться "ИИ заменит менеджеров". Слайд явно показывает что нет. Про вероятности — это снимает страх "а вдруг система ошибётся". Она честно показывает уверенность, менеджер решает.

## Слайд 3 — Система умнеет сама

**Заголовок:** "Каждый новый источник — стратегии точнее"

**Диаграмма:** источники слева + эффекты справа (draw.io, how-to-improve)

**Левая часть — источники:**
```
✅ Сегодня:
  Реестры компаний → кто директор, учредители, структура собственности
  Ваши отчёты и контакты → кто с кем работал, кто ЛПР, личные связи

🔜 Завтра:
  Тендеры и госзакупки → в каких проектах участвовала компания и с кем
  Отраслевые новости → что происходит с компанией прямо сейчас
```

**Правая часть — эффект:**
```
Больше источников → Граф знает больше → AI видит больше паттернов
→ Стратегии входа становятся точнее
```

**Почему граф важнее любого AI:**
- Сырые данные хранятся всегда
- Появится более умный AI — прогоним те же данные заново
- Получим лучший результат без повторного сбора информации

**Ключевая фраза:**
> "Ваши данные не устаревают. Система становится умнее вместе с AI."

**Заметка докладчику:**
Инвестиция в данные не обесценивается. Каждый новый источник автоматически обогащает все существующие проекты — старые проекты тоже получают новые данные.

## Слайд 4 — Безопасность данных

**Заголовок:** "Данные остаются у вас"

**Диаграмма:** три блока — сервер, анонимайзер, AI (draw.io, how-we-defence)

**Как это работает:**
```
Сервер                    Модуль анонимайзер              AI
Реальные имена,  →  AI видит только коды  →  Анализирует граф
контакты             Сервер получает           с кодами
                     реальные имена       ←
```

**Три тезиса:**
- Система работает на вашем сервере — данные никуда не уходят
- AI никогда не видит реальные имена — только зашифрованные коды
- Доступ разработчиков — только по вашему запросу и с вашего ведома

**Ключевая фраза:**
> "ФИО, телефоны, ИНН — никогда не покидают ваш периметр."

**Диаграмма:** система → единый интерфейс → три провайдера (draw.io, how-we-stay-available)

**Три тезиса:**

- Если один провайдер недоступен — переключение без остановки системы
- Архитектура готова к работе с любым AI который появится завтра

**Идея на будущее:**
> "В перспективе — два независимых синхронизирующихся контура: российский и зарубежный."

**Заметка докладчику:** закрывает вопрос "а вдруг заблокируют". Не углубляться в детали.

## Слайд 6 — Как запускаем

**Заголовок:** "Сначала проверяем — потом строим"

**Диаграмма:** четыре этапа (draw.io), PoC выделен зелёной рамкой

```
[PoC]           [MVP]              [V1]               [V2]
Проверка    →   Рабочий        →   Система         →  Система
концепции       инструмент         думает             следит сама
1-2м            6м                 и рекомендует
```

**PoC — главный месседж:**
> "Сначала берём 2–3 ваших реальных проекта, загружаем в систему и показываем что она находит. Только после этого решаем идти дальше."

Что входит в PoC:
- Ваши реальные данные и контакты в системе
- Граф связей по 2–3 проектам
- Ответ на вопрос "через кого зайти в компанию X"

**MVP — рабочий инструмент:**
- Карта связей по всем вашим проектам
- Автоматическое обогащение из реестров компаний
- Базовый отчёт по проекту: участники, ЛПР/ЛВР, экспорт PDF
- Чат-бот для быстрых запросов к графу
- Уведомления в Telegram когда что-то меняется у ваших контактов

**V1 — система думает и рекомендует:**
- Стратегии входа с оценкой риска каждого пути
- Карта рукопожатий в отчёте
- AI объясняет почему рекомендует именно этот путь

**V2 — система следит за рынком сама:**
- Автоматический мониторинг тендеров и госзакупок
- Отслеживание новостей по компаниям из ваших проектов
- Более глубокий AI анализ по накопленным данным

**Ключевая фраза:**
> "Прозрачный старт: результат виден до принятия масштабных решений."

**Заметка докладчику:**
Не называть конкретные сроки MVP/V1/V2. Если директор давит — "нам нужно посмотреть на ваши данные чтобы дать честную оценку."
