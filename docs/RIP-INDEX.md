## Что это за проект

**Заказчик:** строительная фирма, B2B, крупные заказы, мало проектов.

**Суть системы:** корпоративная память о связях между людьми и компаниями, оформленная в граф. Не CRM — граф рукопожатий. Главная задача: быстро найти оптимальную стратегию входа в новый проект через уже известные контакты (ЛПР/ЛВР).

**Кто разрабатывает:** мы (мощные фрилансеры), стек — DCA фреймворк на TypeScript/NestJS + Python микросервис для AI.

**Текущий статус:** подготовка к первому созвону с директором. Все документы — черновики до подписания договора.

## Ключевые решения — уже приняты

**Архитектура:**
- Граф на PostgreSQL + recursive CTE (не Neo4j, критерий перехода: >500ms на shortest_path)
- Граф двунаправленный — "А знает Б" = "Б знает А"
- Контакты расшарены между всеми менеджерами компании
- Поиск пути стартует от всех менеджеров одновременно (multi-source)
- Два микросервиса: NestJS (основной) + FastAPI Python (AI/extraction)
- Job queue на PostgreSQL в MVP → Redis+BullMQ в v2
- DCA фреймворк как основа разработки

**Граф и поиск пути:**
- shortest_path + weighted_path — система показывает оба варианта
- Weighted path: вес = relationship_type_score × strength × direction_penalty
- Друзья и партнёры — высокий вес. Формальная иерархия снизу вверх — низкий вес
- Длина пути ≠ качество пути

**Данные:**
- Data pipeline: Observation → Signal → Fact → Snapshot
- Версионирование: temporal tables (valid_from / valid_to), ничего не удаляется
- Raw data хранится всегда — для возможного переобучения без повторного скрапинга
- ИНН как anchor для entity resolution компаний
- Типы связей: friends, ex_colleagues, partners, colleagues, works_at, worked_at, affiliated_with

**AI и безопасность:**
- Pseudonymization per-project: токены фиксированы на уровне проекта, маппинг в БД
- LLM Connector — абстракция провайдера (Ollama / GigaChat в MVP)
- Source Connector — абстракция источников данных (DaData в MVP)
- Оба коннектора через единый интерфейс — главная долгосрочная ценность системы
- Метаданные графа — заготовка под AI анализ в v3

**MVP скоуп:**
- Онбординг: разработчики вносят данные вручную, менеджер верифицирует
- AI Extraction из PDF — не в MVP, опционально позже
- Чатбот: 8 команд через intent matching (не свободный диалог)
- Раздел "Стратегии входа" в Project Report — AI анализирует граф и даёт 2-3 варианта с reasoning и ссылками на источники. Это часть Snapshot Worker, не отдельный модуль.
- Refresh Worker: ночной cron, DaData, Telegram уведомления
- Outlook: CSV импорт телефонов и имён при онбординге (разовый скрипт). Парсинг переписки не планируется.

**Запуск:**
- Этапы: PoC (проверка) → MVP (рабочий инструмент) → V1 (умная аналитика) → V2 (автоматизация)
- PoC: 2-3 недели, отдельный контракт, фиксированная стоимость
- MVP: граф + чатбот + уведомления + базовый PDF отчёт
- V1: стратегии входа с AI reasoning + Relationship Map + Feedback
- V2: тендеры, новости, Rank Worker, интерактивный граф
- Никогда не показывать систему на пустой базе
